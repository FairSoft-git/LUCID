# Documentation Structure

> **Owner:** LF

This directory contains all project documentation.

## Structure

- **development/**: Technical guides, setup instructions, and architecture decisions.
- **education/**: User guides, tutorials, and domain knowledge.
- **api/**: API references and interface specifications.

## Contributing

When adding new documentation:
1. Use the templates in the project (e.g., `docs/`, `features/`, `tests/`, `scripts/`).
2. Keep documents focused and concise (LUCID philosophy).
3. Update the index below if adding a new top-level section.
